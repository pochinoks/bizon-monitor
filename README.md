# BizonMonitor

Автоматический мониторинг чата вебинаров Bizon365 — сообщения участников приходят в Telegram.

## Как работает

Каждый день GitHub Actions запускает скрипт по расписанию:
- **15:58 UTC** (NY 12:00 летом) — подключается к 4 комнатам параллельно
- **22:58 UTC** (NY 19:00 летом) — подключается к `ai_extra_income`

Слушает 2 часа, все сообщения гостей летят в Telegram. Логин автоматический — ничего вручную делать не нужно.

## Secrets (Settings → Secrets → Actions)

| Secret | Что это |
|--------|---------|
| `BIZON_EMAIL` | Email от аккаунта Bizon365 |
| `BIZON_PASSWORD` | Пароль от аккаунта Bizon365 |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram бота |
| `TELEGRAM_CHAT_ID` | ID чата/группы для уведомлений |

## Ручной запуск для теста

Actions → Webinar 16:00 UTC → Run workflow

## Смена сезона

Зимой (EST) NY 12:00 = UTC **17:00**, NY 19:00 = UTC **00:00**.  
В `.github/workflows/webinar-16.yml` поменяй cron: `58 15` → `58 16`  
В `.github/workflows/webinar-23.yml` поменяй cron: `58 22` → `58 23`

## Комнаты

Настраиваются в `index.js` → `CONFIG.WEBINARS`.

## Почему репо публичное

GitHub даёт **неограниченные минуты** Actions только для публичных репо.  
Для приватных — лимит 2000 мин/месяц, а скрипт потребляет ~7000 мин/месяц.  
Все чувствительные данные хранятся в Secrets — в коде их нет.
